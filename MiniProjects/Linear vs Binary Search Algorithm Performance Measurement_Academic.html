# linear search function
def lin_search(arr, tgt):
    for i in arr:
        if i == tgt:
            return np.where(arr == tgt)
    return None

# binary search function
def binary_search(arr, tgt):
    low = 0
    high = len(arr) - 1
    while low <= high:
        mid = math.floor((low + high) / 2)
        guess = arr[mid]
        if guess == tgt:
            return mid
        if guess > tgt:
            high = mid - 1
        else: 
            low = mid + 1
    return None

# loop through each array and gather performance data
for item in arrays:
    length = len(item)
    lengths.append(length)
    start1 = timer()
    item.sort()
    end1 = timer()
    sort_speed = (end1 - start1) * 1000
    sort_times.append(sort_speed)
    tgt = max(item)
    start2 = timer()
    lin_search(item, tgt)
    end2 = timer()
    lin_srch_speed = (end2 - start2) * 1000
    lin_srch_times.append(lin_srch_speed)
    start3 = timer()
    binary_search(item, tgt)
    end3 = timer()
    bin_srch_speed = (end3 - start3) * 1000
    bin_srch_times.append(bin_srch_speed)

Executive Summary¶

In this experiment we tested the built-in sort function, user defined linear search function, and user defined binary search function. I have provided a data table (dataframe) that provides the processing speed results of each operation in milliseconds. The scatterplots below the data table also visualize the results of the experiment.

The table and visualizations demonstrate the improvement in search speed for the binary search function as opposed to the linear search function. While the time increases as the array size increases, the difference in search times between the linear function and the binary function are significant, especially as the array size increases. In the largest array of 704 items, the linear search time is ~.28 milliseconds while the binary search time is ~.04 milliseconds. In contrast, referring to the shortest array with 125 items, the linear search time is ~.06 milliseconds while the binary search time is ~.02 milliseconds. This variance in performance captures how as the array grows in size, the performance gap of each search algorithm drastically favors the binary search. As for the sort function, the processing speed grows in a relatively linear fashion as the size of the array increases.

In all cases the processing speed has a positive linear relationship based on the array size. For the combination measurements, linear search time plus sort time, and binary search time plus sort time, the search times and sort times were added together in each instance. Again, the binary search instances outperform the linear search instances.

The binary search function speeds up the process of finding a target value in the array by cutting the array in half as it searches; it partitions the data (after being sorted) based on a greater than or less than method to more quickly find the target. It begins from an initial split, then a new split within the initial split, and so on until it has honed in on the target variable.

In BigO notation, the binary search function is measured as O(log n); the natural logarithm of n, with n representing the length of the array. This is also known as log time. As for the speed of the linear search function, it is measured as O(n), or linear time; the speed is based on the length of the array. The sort() function is a built in Python function. In BigO notation, the sort function is measured as O(n log n). This is a fast sorting algorithm.

The binary search function is useful to data engineers as they can optimize data processing functions at a faster rate, as opposed to linear time. The processing time for large amounts of data using linear time techniques would severely slow down our systems as the data increases. The processing time for large amounts of data using a log time technique increases its advantage over a linear time technique as the data grows.
